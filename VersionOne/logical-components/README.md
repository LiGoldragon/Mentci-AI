# Logical Components (VersionOne)

This tree tracks architecture units by maturity status.

## Status Buckets
- `selected/` — approved and prioritized for implementation.
- `in-review/` — currently in deliberation or refinement.
- `complete/` — accepted v1 logic units with stable contract.

## Selection Criteria
A component is considered `selected` when:
1. purpose and boundaries are clearly defined,
2. message contract surface is specified,
3. lane-policy alignment is confirmed.

## Completion Criteria
A component is considered `complete` when:
1. governance and data contract are stable,
2. operational behavior has no open review blockers,
3. status is reflected in structured data sidecars.
